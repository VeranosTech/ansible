YAML 문법¶

이 페이지는 앤서블 플레이북에(그리고 설정 관리 언어로) 쓰이는 YAML의 올바른 문법을 설명한다.

YAML이 XML이나 JSON과 같은 다른 데이터 형식보다 사람이 읽고 쓰기 쉽기 때문에 우리는 YAML을 사용한다. 또한 YAML을 다룰 수 있는 라이브러리는 대부분의 프로그래밍 언어에서 제공한다.

실제로 YAML이 어떻게 쓰이는지를 보려면 플레이북 작업도 같이 읽으면 좋다.

YAML 기초¶

앤서블에서는 거의 모든 YAML 파일이 리스트로 시작한다. 리스트의 각 아이템은 "해시" 또는 "딕셔너리"로 불리는 키/밸류 쌍의 리스트이다. 따라서 우선 YAML에서 리스트와 딕셔너리를 쓰는 법을 알아야 한다.

YAML에는 또다른 특징이 있는데 모든 YAML 파일은 (앤서블과 상관없이) ---로 시작할 수 있고 ...로 끝날 수 있다. 이 것은 YAML 형식의 일부로서 문서의 시작과 끝을 나타낸다.

리스트의 모든 원소는 같은 들여쓰기이고 "- " (대시와 공백)으로 시작한다:

---
# A list of tasty fruits
- Apple
- Orange
- Strawberry
- Mango
...

딕셔너리는 key: value 형식이다. (콜론 뒤에 반드시 공백이 있어야 한다.):

# An employee record
martin:
    name: Martin D'vloper
    job: Developer
    skill: Elite

딕셔너리의 리스트라든가 리스트를 원소로 가지는 딕셔너처럼 더 복잡한 자료구조도 가능하다.

# Employee records
-  martin:
    name: Martin D'vloper
    job: Developer
    skills:
      - python
      - perl
      - pascal
-  tabitha:
    name: Tabitha Bitumen
    job: Developer
    skills:
      - lisp
      - fortran
      - erlang

딕셔너리와 리스트는 다음처럼 단축형식으로 쓸 수도 있다.:

---
martin: {name: Martin D'vloper, job: Developer, skill: Elite}
['Apple', 'Orange', 'Strawberry', 'Mango']

이를 "플로우 콜렉션(flow collection)"이라고 부른다.

앤서블에서는 잘 쓰이지 않지만 불리언 값도 여러가지 형식으로 지정할 수 있다.:

create_key: yes
needs_agent: no
knows_oop: True
likes_emacs: TRUE
uses_cvs: false

값을 여러줄에 걸처 쓰러면 | 또는 >를 사용한다. 여러줄에 걸처 "리터럴 블럭 스칼라(Literal Block Scalar)" |를 쓰면 줄넘기기와 뒤쪽 공백도 포함하게 된다. "폴디드 블럭 스칼라(Folded Block Scalar)" >를 사용하면 줄넘기기와 뒤쪽 공백은 없어진다. 원래 한 줄인 것을 읽거나 편집하기 편하게 나눌 때 사용한다. 어느 경우든 들여쓰기는 무시된다. 예제는 다음과 같다.:

include_newlines: |
            exactly as you see
            will appear these three
            lines of poetry

fold_newlines: >
            this is really a
            single line of text
            despite appearances

앞에서 > 예제는 모든 줄넘기기가 공백으로 변환되었다. 줄넘기기를 유지하려면 두 가지 방법이 있다.:

fold_some_newlines: >
    a
    b

    c
    d
      e
    f
same_as: "a b\nc d\n  e\nf\n"

지금까지 배운것을 종합하여 임의의 YAML을 만들어보자. 앤서블과는 관계없지만 YAML 형식에 대해 감이 올 것이다.:

---
# An employee record
name: Martin D'vloper
job: Developer
skill: Elite
employed: True
foods:
    - Apple
    - Orange
    - Strawberry
    - Mango
languages:
    perl: Elite
    python: Elite
    pascal: Lame
education: |
    4 GCSEs
    3 A-Levels
    BSc in the Internet of Things

이게 앤서블 플레이북을 만들기 위해 필요한 YAML의 전부이다.

Gotchas¶

따옴표가 없는 스칼라값으로 넣을 수 없는 것이 몇가지 있다. 콜론 뒤에 공백(또는 줄바꿈)이오는 ": "는 딕셔너리에 쓰이는 기호이다. 공백다음에 파운드 기호 " #"도 주석(comment)의 시작이다.

이것 때문에 다음 코드는 YAML 문법 오류가 나온다.:

foo: somebody said I should put a colon here: so I did

windows_drive: c:

하지만 다음 코드는 동작한다.:

windows_path: c:\windows

해시 값이 되려면 콜론 다음에 공백이나 줄바꿈을 사용한다.:

foo: 'somebody said I should put a colon here: so I did'

windows_drive: 'c:'

그러면 콜론이 남아있을 수 있다.

아니면 겹따옴표를 쓸 수 있다.:

foo: "somebody said I should put a colon here: so I did"

windows_drive: "c:"

홑따옴표와 겹따옴표의 차이는 겹따옴표에서는 이스케이프(escape)가 된다는 것이다.:

foo: "a \t TAB and a \n NEWLINE"

"Escape Sequences" (YAML 1.1) 또는 "Escape Characters" (YAML 1.2) YAML 표준에 따르면 허용되는 이스케이프 문자는 다음과 같다.

The following is invalid YAML:

foo: "an escaped \' single quote"

또 앤서블에서는 변수에 대해 "{{ var }}"를 쓸 수 있다. 만약 콜론 다음의 값이 "{"로 시작하면 YAML은 이를 딕셔너리로 인식한다. 따라서 다음처럼 따옴표가 있어야 한다.:

foo: "{{ variable }}"

만약 값이 따옴표로 시작하면 값의 일부가 아니라 전체에 대해 따옴표를 붙여야 한다.다음은 따옴표의 올바른 사용법이다.:

foo: "{{ variable }}/additional/string/literal"
foo2: "{{ variable }}\\backslashes\\are\\also\\special\\characters"
foo3: "even if it's just a string literal it must all be quoted"

잘못된 코드의 예

foo: "E:\\path\\"rest\\of\\path

'와 " 이외에도 특별이 예약된 문자가 있다. 이글자들은 따옴표가 없은 스칼라값의 첫 글자로 오지 못한다.

? : -에 주의하라. 만약 공백문자가 아닌 클자가 붙으면 문자열의 시작에 쓸 수 있다. 하지만 YAML 처리기는 다르게 구현할 수 있기 때문에 함부로 인용하면 안된다.

"플로우 콜렉션"에서의 규칙은 좀 더 엄격하다

a scalar in block mapping: this } is [ all , valid

flow mapping: { key: "you { should [ use , quotes here" }

불리언 변환이 도움이 되지만 문자로 된 리터럴 yes 또는 다른 불리언값을 원할 수도 있다.:

non_boolean: "yes"
other_string: "False"

YAML은 1.0등의 특정한 문자열을 부동소수점으로 변환한다. 만약 (예를 들어 requirements.ym 파일에서처럼) 버전 넘버를 표시해야 하면 따옴표로 인용해야 한다. 아니면 정말 부동소수점인 것처럼 변환한다

version: "1.0"
