ansible¶

'playbook'이라는 단일 작업을 정의하고 지정된 호스트 그룹에서 실행시킨다.

개요¶

usage: ansible [-h] [--version] [-v] [-b] [--become-method BECOME_METHOD]
            [--become-user BECOME_USER] [-K] [-i INVENTORY] [--list-hosts]
            [-l SUBSET] [-P POLL_INTERVAL] [-B SECONDS] [-o] [-t TREE] [-k]
            [--private-key PRIVATE_KEY_FILE] [-u REMOTE_USER]
            [-c CONNECTION] [-T TIMEOUT]
            [--ssh-common-args SSH_COMMON_ARGS]
            [--sftp-extra-args SFTP_EXTRA_ARGS]
            [--scp-extra-args SCP_EXTRA_ARGS]
            [--ssh-extra-args SSH_EXTRA_ARGS] [-C] [--syntax-check] [-D]
            [-e EXTRA_VARS] [--vault-id VAULT_IDS]
            [--ask-vault-pass | --vault-password-file VAULT_PASSWORD_FILES]
            [-f FORKS] [-M MODULE_PATH] [--playbook-dir BASEDIR]
            [-a MODULE_ARGS] [-m MODULE_NAME]
            pattern

설명¶

'원격 작업'을 하기 위한 간단한 도구/프레임워크/API이다. 이 명령을 사용하면 'playbook'이라는 단일 작업을 정의하고 지정된 호스트 그룹에서 실행 시킬 수 있다.

공통 옵션¶

--ask-vault-pass¶

valut 패스워드를 묻는다.

--become-method <BECOME_METHOD>¶

privilege escalation method to use (default=%(default)s), use ansible-doc -t become -l to list valid choices.

--become-user <BECOME_USER>¶

이 사용자로 명령 실행 (디폴트는 root)

--list-hosts¶

해당되는 호스트 목록 출력. 그외는 아무것도 하지 않는다.

--playbook-dir <BASEDIR>¶

Since this tool does not use playbooks, use this as a substitute playbook directory.This sets the relative path for many features including roles/ group_vars/ etc.

--private-key <PRIVATE_KEY_FILE>, --key-file <PRIVATE_KEY_FILE>¶

이 파일로 접속 인증

--scp-extra-args <SCP_EXTRA_ARGS>¶

scp에만 전달하는 추가 인수 지정

--sftp-extra-args <SFTP_EXTRA_ARGS>¶

sftp에만 전달하는 추가 인수 지정

--ssh-common-args <SSH_COMMON_ARGS>¶

sftp/scp/ssh에 전달하는 공통 인수 지정

--ssh-extra-args <SSH_EXTRA_ARGS>¶

ssh에만 전달하는 추가 인수 지정

--syntax-check¶

실행하지 않고 문법만 확인한다.

--vault-id¶

사용할 valut 아이디 지정

--vault-password-file¶

vault 패스워드 파일

--version¶

프로그램의 버전, 설정 파일 위치, 모듈 검색 패스, 모듈 위치, 실행 위치 등을 보여준다.

-B <SECONDS>, --background <SECONDS>¶

비동기적 실행. X 초후에 실패반환 (디폴트 없음)

-C, --check¶

실제로 실행하지 않고 어떤 것이 바뀌는지만 보인다.

-D, --diff¶

파일과 템플릿이 변화되면 변화된 부분을 보여준다. --check와 같이 사용하면 좋다.

-K, --ask-become-pass¶

권한상승용 패스워드를 묻는다.

-M, --module-path¶

prepend colon-separated path(s) to module library (default=~/.ansible/plugins/modules:/usr/share/ansible/plugins/modules)

-P <POLL_INTERVAL>, --poll <POLL_INTERVAL>¶

-B 옵션을 사용할 경우 폴링 간격을 걸정한다. (디폴트 15)

-T <TIMEOUT>, --timeout <TIMEOUT>¶

override the connection timeout in seconds (default=10)

-a <MODULE_ARGS>, --args <MODULE_ARGS>¶

모듈 인수

-b, --become¶

become으로 명령 실행 (패스워드를 묻지 않는다)

-c <CONNECTION>, --connection <CONNECTION>¶

사용할 접속 유형 (디폴트는 smart)

-e, --extra-vars¶

key=value 형태로 추가 변수 설정. 만약 filename이 @로 시작하면 YAML/JSON 사용

-f <FORKS>, --forks <FORKS>¶

동시에 사용할 병렬 프로세스 수 지정 (디폴트 15)

-h, --help¶

이 도움말을 보여준다.

-i, --inventory, --inventory-file¶

인벤토리 호스트 경로 또는 콤마로 분리된 호스트 리스트를 지정. --inventory-file 옵션은 없어짐.

-k, --ask-pass¶

접속 패스워드를 묻는다.

-l <SUBSET>, --limit <SUBSET>¶

추가적인 패턴으로 호스트를 제한한다.

-m <MODULE_NAME>, --module-name <MODULE_NAME>¶

실행할 모듈 이름 (디폴트는 command)

-o, --one-line¶

압축된 출력

-t <TREE>, --tree <TREE>¶

이 디렉토리에 출력 로그 생성

-u <REMOTE_USER>, --user <REMOTE_USER>¶

지정 사용자로 접속 (디폴트 없음)

-v, --verbose¶

verbose 모드 (-vvv는 설명이 더 많다. -vvvv 는 접속 디버깅이 가능)

환경¶

다음 환경변수를 설정할 수 있다.

ANSIBLE_CONFIG -- Ansible 설정 파일의 값을 덮어쓴다.

ansible.cfg에는 더 많은 옵션을 사용할 수 있다.

파일¶

/etc/ansible/ansible.cfg -- 설정파일. 있으면 사용

~/.ansible.cfg -- 사용자 설정 파일, 존재하면 기본 설정 파일에 우선
